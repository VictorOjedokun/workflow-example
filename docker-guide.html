Beginner Guide

Docker from Zero

Build, run, and ship a Node.js app inside a container — step by step.

📁 Your project files

📄
app.js
The Node.js web server (your actual app)
📦
package.json
Lists your app's dependencies (express)
🐳
Dockerfile
Recipe Docker uses to build your image
🚫
.dockerignore
Files Docker should NOT copy into the image
1

Install Docker Desktop

Download and install Docker Desktop for your OS. It includes everything you need.

https://www.docker.com/products/docker-desktop/

After installing, verify it works:

$ docker --version # Expected output: Docker version 26.x.x, build ...
2

Set up your project folder

Create a folder and place all four files inside it.

$ mkdir my-docker-app $ cd my-docker-app # Now copy app.js, package.json, Dockerfile, .dockerignore here

Your folder should look like this:

my-docker-app/ ├── app.js ├── package.json ├── Dockerfile └── .dockerignore
3

Build the Docker image

This reads your Dockerfile and packages everything into an image. Run this from inside your project folder.

$ docker build -t my-app . # -t my-app → name/tag your image "my-app" # . → use the current directory (where Dockerfile is)
⏳ First time? Docker downloads the Node.js base image (~200 MB). This only happens once — future builds are much faster.

When it finishes you'll see:

=> => writing image sha256:abc123... => => naming to docker.io/library/my-app
4

Run the container

Start a container from your image and expose port 3000 so you can reach it in your browser.

$ docker run -p 3000:3000 my-app # -p 3000:3000 → map port 3000 on your machine to port 3000 inside the container
✅ You should see: Server running on http://localhost:3000

Open your browser and visit http://localhost:3000 — your app is live!

5

Stop the container

Press Ctrl + C in the terminal to stop it. To run it in the background instead:

$ docker run -d -p 3000:3000 --name myapp my-app # -d → detached mode (runs in background) # --name myapp → give the running container a friendly name $ docker stop myapp # stop it later $ docker start myapp # start it again

💡 Key concepts in plain English

Image — A snapshot/blueprint of your app and everything it needs. Like a ZIP file you share with others.

Container — A running instance of an image. Like opening that ZIP file and actually running it.

Dockerfile — A text recipe that tells Docker how to build your image, step by step.

Port mapping (-p) — Your laptop and the container each have their own "doors" (ports). -p 3000:3000 connects your door to the container's door.
